An AX designer shapes what AI agents do — not just what screens look like.

If a UX designer's question is "How does the user interact with this system?", an AX designer's question is "What does the agent do on its own, and how does the user experience that?"

The difference matters because agentic AI is changing the fundamental unit of design from interface to behavior. Here's what that actually means in practice.

## A Day in the Life of an AX Designer

There's no single day — it depends on the product stage. But most AX designers split their time across four types of work.

### 1. Writing Agent Behavior Specs (40%)

This is the core of the job. You define how the agent behaves through structured instructions — not vague prompts, but systematic behavior specifications.

A behavior spec includes:
- **Goal definition:** What the agent is trying to achieve
- **Decision criteria:** How the agent prioritizes options
- **Constraints:** What the agent must never do
- **Fallback behavior:** What happens when the agent is uncertain or fails

Example: You're designing a travel planning agent. The spec defines: "When the user provides a budget and destination, search flights within ±10% of the budget, prioritize direct flights, flag options that exceed budget by more than 15%, and present results sorted by best value (not cheapest)."

### 2. Designing Conversation Flows (25%)

Every agent communicates through language. You design those conversations.

A conversation flow defines:
- **How the agent asks for clarification** when the user's request is ambiguous
- **How the agent presents uncertainty** — "I think this is right" vs "This is correct"
- **How the agent handles errors** — "I couldn't find X. Should I try Y instead?"
- **How the agent escalates** to human review when needed

This isn't UX writing. It's interaction design through language.

### 3. Testing and Iteration (25%)

You don't just design and hand off. You test agent behavior, analyze failures, and iterate.

This involves:
- Running the agent through 50+ scenario tests
- Analyzing patterns in agent mistakes
- Identifying edge cases that produce bad outputs
- Refining behavior specs based on results
- Measuring success by trust and accuracy, not just task completion

### 4. Collaboration (10%)

You work with:
- **ML engineers** who build the agent architecture
- **Product managers** who define the agent's business goals
- **UX researchers** who study how humans interact with the agent
- **Domain experts** who understand the knowledge the agent works with

## How AX Design Differs from Product Design

The overlap is significant. Both roles care about user needs, iteration, and outcomes. But the focus differs:

| Product Designer | AX Designer |
|---|---|
| Designs screens and flows | Designs agent behavior |
| User directs the interaction | Agent acts autonomously |
| Tests wireframes and prototypes | Tests agent outputs in scenarios |
| Measures task completion | Measures trust and accuracy |
| Primary skill: visual + interaction design | Primary skill: prompt + conversation design |
| Artifacts: Figma files | Artifacts: behavior specs |

## The Skills You Need

You don't need to learn to code. You need three skills:

1. **Prompt design.** Writing structured instructions that reliably guide AI behavior. This is not "write a prompt." This is designing behavior specifications with constraints, examples, and fallback rules.
2. **Conversation design.** Designing the dialogue between user and agent. How does the agent communicate uncertainty? How does it recover from errors? How does it explain its reasoning?
3. **Systems thinking.** Understanding how AI agents perceive, plan, act, and iterate — and designing for that full cycle of autonomous behavior.

These build on your existing UX foundation. Your empathy, your understanding of user needs, your ability to think about edge cases — those transfer directly.

## Where the Role Is Headed

As AI agents move from novelty to infrastructure, the AX designer role will split into specialties:

- **Agent Behavior Designer:** Focuses on decision-making, planning logic, and action selection
- **Conversation Designer:** Focuses on dialogue patterns, tone, and communication
- **Trust Designer:** Focuses on transparency, confidence signals, and error recovery
- **Agent UX Designer:** Focuses on the full human-agent experience

These aren't separate people — they're different lenses on the same role. The best AX designers can do all four.

## Frequently Asked Questions

### What does an AX designer do on a typical day?

An AX designer writes agent behavior specs, designs conversation flows, tests agent outputs against scenarios, and iterates based on results. You spend your time defining how agents reason, communicate, and handle uncertainty — not drawing screens.

### How is an AX designer different from a product designer?

A product designer focuses on interfaces, flows, and user tasks. An AX designer focuses on agent behavior, decision-making, and autonomous action. Product designers design what the user sees. AX designers design what the system does.

### Do AX designers need to code?

No. You need technical literacy — understanding how agents work conceptually — but not production coding. You write prompts, not programs. You design behavior, not interfaces.

### What career path leads to AX design?

Most AX designers come from product design, UX research, or conversation design backgrounds. The transition takes 2–3 months of focused learning in prompt design, conversation design, and systems thinking for AI.

### What skills are most important for an AX designer?

Prompt design (writing reliable agent instructions), conversation design (designing dialogue patterns), and systems thinking (understanding how agents perceive, plan, act, and iterate). Your existing UX empathy is the foundation.
