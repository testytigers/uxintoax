## The Short Answer

Agent behavior design is about defining how AI agents behave — what they do, when they do it, how they respond, and what constraints govern their actions. The right amount of autonomy depends on risk level. Low-risk agents can be autonomous; high-risk agents require user confirmation.

---

## The Core Framework

### 1. Define the Agent's Purpose

What is the agent trying to achieve? Clear purpose drives clear behavior.

**Examples:**
- "This agent helps users organize their email inbox."
- "This agent helps users create presentation slides."
- "This agent helps users find relevant research papers."

### 2. Define the Agent's Scope

What is the agent allowed to do? What is it not allowed to do?

**Rules:**
- List specific allowed actions
- List specific prohibited actions
- Define boundaries and constraints
- Specify when the agent should ask for user input

### 3. Define the Agent's Decision Logic

How does the agent decide what to do?

**Options:**
- Rule-based (if X, then Y)
- Learning-based (based on patterns in user behavior)
- Hybrid (rules for high-stakes, learning for low-stakes)

### 4. Define the Agent's Communication Style

How does the agent communicate with users?

**Elements:**
- Tone (formal, casual, professional, friendly)
- Length (concise, detailed, adaptive)
- Style (direct, conversational, instructional)

### 5. Define the Agent's Constraints

What limits the agent's behavior?

**Types:**
- Safety constraints (cannot perform certain actions)
- Privacy constraints (cannot access certain data)
- Temporal constraints (can only act during certain hours)
- Quantity constraints (can only perform a limited number of actions)

---

## Autonomy Levels

| Level | Description | Examples |
|---|---|---|
| L1: Assist | Agent suggests, user decides | ChatGPT responses, design suggestions |
| L2: Collaborate | Agent acts, user confirms | Email drafts that need review |
| L3: Delegate | Agent acts, user can review | Auto-filing emails, auto-scheduling meetings |
| L4: Autonomy | Agent acts without prompting | Auto-responding to routine emails |

---

## The Bottom Line

Agent behavior design is about defining what agents do, when they do it, and how they communicate. Start with clear purpose, define scope and constraints, and set the right level of autonomy based on risk.
