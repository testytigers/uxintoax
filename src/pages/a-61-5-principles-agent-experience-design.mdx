## The Short Answer

The five foundational principles of Agent Experience (AX) design are: transparency, user control, predictability, accountability, and empathy. User control is the most important — without it, users lose trust. Transparency (showing what the agent is doing), predictability (letting users anticipate behavior), accountability (clarifying responsibility), and empathy (respecting user feelings) complete the framework.

---

## Principle 1: Transparency

Users must understand what the agent is doing and why.

**Why it matters:** AI agents operate in the background. Without transparency, users don't know what's happening, which creates anxiety and distrust.

**How to design it:**
- Show the agent's current task and progress
- Explain why the agent chose a particular action
- Reveal the agent's reasoning (when possible)
- Make the agent's capabilities and limitations visible

**Example:**
Bad: An email agent auto-drafts responses without explanation.
Good: An email agent shows its draft with a note: "I suggested this response because you mentioned the deadline is Friday."

---

## Principle 2: User Control

Users must be able to override, modify, or shut down the agent at any time.

**Why it matters:** AI agents can make mistakes. Without control, users feel trapped and lose trust.

**How to design it:**
- Provide clear override mechanisms
- Make it easy to undo agent actions
- Give users granular control over what the agent can do
- Allow users to set boundaries (what the agent can and cannot do)

**Example:**
Bad: An agent deletes emails without asking.
Good: An agent asks: "Should I delete these 5 spam emails?" with Yes/No buttons.

---

## Principle 3: Predictability

Users must be able to anticipate what the agent will do.

**Why it matters:** AI agents are probabilistic — their output varies. Without predictability, users can't rely on the agent.

**How to design it:**
- Show consistent behavior patterns
- Provide examples of what the agent typically does
- Set clear expectations about what the agent can and can't do
- Use consistent language and interaction patterns

**Example:**
Bad: An agent sometimes responds with one word, sometimes with a paragraph.
Good: An agent consistently provides structured, appropriately-length responses based on the request.

---

## Principle 4: Accountability

Users must know who is responsible when the agent does something wrong.

**Why it matters:** AI agents can produce harmful or incorrect output. Without accountability, users don't know who to hold responsible.

**How to design it:**
- Clearly state that the agent is an AI and the user is responsible
- Provide clear channels for reporting agent errors
- Document agent decisions and actions
- Make it clear who created the agent and who supports it

**Example:**
Bad: An agent makes a financial recommendation that loses money. No one is responsible.
Good: An agent states: "This is AI-generated advice. I'm not a financial advisor. Consult a professional for investment decisions."

---

## Principle 5: Empathy

Agents must communicate in a way that respects user feelings and context.

**Why it matters:** AI interactions are conversational. Poorly designed agents can feel cold, dismissive, or even insulting.

**How to design it:**
- Use appropriate tone and language
- Acknowledge user frustration when the agent makes mistakes
- Avoid overly casual or overly formal language
- Adapt the agent's tone to the user's emotional state

**Example:**
Bad: An agent says "LOL, I can't do that." when it can't complete a task.
Good: An agent says "I'm sorry, I can't complete that task. Here's what I can do instead."

---

## How the Principles Work Together

The five principles are interconnected:

- **Transparency** enables **predictability** — when users understand what the agent is doing, they can anticipate its behavior.
- **User control** enables **accountability** — when users can override the agent, they know who is ultimately responsible.
- **Empathy** enables **transparency** — when the agent communicates respectfully, users are more likely to understand and accept what it's doing.
- **Accountability** enables **user control** — when users know who is responsible, they feel empowered to take control.

---

## The Bottom Line

Agent Experience design requires five principles: transparency, user control, predictability, accountability, and empathy. User control is the most important — without it, users lose trust. These principles are interconnected and must be applied together.

If you're designing an AI agent, start with user control. Then add transparency, predictability, accountability, and empathy. These five principles are the foundation of good AX design.
