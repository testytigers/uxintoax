## The Short Answer

Save these 15 AI prompts for your next project. They cover the tasks you repeat every week — research analysis, design brainstorming, content generation, and user testing. AI tools can dramatically speed up these tasks, but only if you ask the right questions.

---

## Research Prompts

### 1. Research Synthesis

```
Analyze these interview transcripts and extract: (1) top 5 themes with supporting quotes, (2) recurring pain points, (3) user goals that are unmet, (4) behavioral patterns. Present as a structured table.

[Insert transcripts]
```

### 2. Survey Response Analysis

```
Analyze these survey responses and provide: (1) sentiment distribution, (2) top 3 complaints, (3) top 3 praises, (4) actionable recommendations. Format as bullet points.

[Insert survey responses]
```

### 3. Competitive Analysis Framework

```
Compare these 3 competitors across: (1) user experience strengths, (2) user experience weaknesses, (3) opportunities for differentiation, (4) UX patterns to adopt or avoid. Format as a comparison table.

[Insert competitor descriptions]
```

### 4. User Persona Generation

```
Create a user persona based on these research findings: (1) demographics, (2) goals and motivations, (3) pain points, (4) behaviors, (5) quote that captures their attitude. Make it specific and actionable, not generic.

[Insert research findings]
```

---

## Design Prompts

### 5. Design Critique

```
Critique this design description and provide: (1) what works well, (2) what could be improved, (3) specific suggestions, (4) UX principles that apply. Be direct and specific.

[Insert design description]
```

### 6. Interaction Flow Ideas

```
Generate 5 interaction flow options for this user task: [Describe task]. For each option, include: (1) step-by-step flow, (2) pros and cons, (3) complexity level (simple, medium, complex).

[Describe task]
```

### 7. Content and Microcopy

```
Write interface text for this [button/error message/help text/navigation item]. Provide 3 variations: (1) clear and direct, (2) friendly and conversational, (3) professional and concise.

[Describe context]
```

---

## Testing Prompts

### 8. Usability Test Script

```
Create a usability test script for testing [product/feature]. Include: (1) 5 tasks to complete, (2) questions to ask after each task, (3) success criteria, (4) scoring rubric. Make tasks realistic and sequential.

[Describe product/feature]
```

### 9. A/B Test Ideas

```
Generate 5 A/B test ideas for improving [metric: e.g., conversion rate, engagement, retention]. For each idea, include: (1) hypothesis, (2) what to test, (3) expected impact, (4) risk level.

[Describe metric and context]
```

---

## Strategy Prompts

### 10. User Journey Mapping

```
Create a user journey map for [user type] doing [task] in [context]. Include: (1) stages, (2) actions at each stage, (3) emotions at each stage, (4) pain points, (5) opportunities for improvement.

[Describe user, task, context]
```

### 11. Information Architecture

```
Suggest an information architecture for this [product type] with: (1) top-level navigation, (2) sub-navigation for each section, (3) content grouping principles. Make it intuitive and user-focused.

[Describe product]
```

### 12. UX Audit Framework

```
Create a UX audit checklist for this [product/feature]. Include categories for: (1) usability, (2) visual design, (3) accessibility, (4) performance, (5) content, (6) interaction. For each category, list 3 specific things to check.

[Describe product]
```

---

## Communication Prompts

### 13. Design Rationale

```
Write a design rationale for this decision: [Describe decision]. Include: (1) the problem, (2) options considered, (3) why we chose this option, (4) expected impact, (5) how we'll measure success.

[Describe decision]
```

### 14. Stakeholder Presentation

```
Create a 5-slide presentation outline for presenting this [design/research/findings] to stakeholders. Each slide should include: (1) title, (2) key message, (3) supporting evidence, (4) call to action.

[Describe what you're presenting]
```

### 15. PRD Summary

```
Summarize this product requirements document for designers. Extract: (1) user problems to solve, (2) success criteria, (3) constraints, (4) target users, (5) key features. Keep it under 300 words.

[Insert PRD]
```

## The Bottom Line

Save these prompts. They cover the tasks you'll repeat every week — research analysis, design brainstorming, content generation, and user testing. The key to getting great results is being specific about your context and iterating on the output.
