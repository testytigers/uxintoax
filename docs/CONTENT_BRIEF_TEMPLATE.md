## 10. FAQ Section

**Format:** 4–6 questions and answers. Each question is a natural-language search query. Each answer is 40–80 words. Written for AEO citation.

**Example:**
```
Q: Is AX the same as UX?
A: No. UX focuses on human-computer interaction — screens, flows, and interfaces. AX focuses on human-agent interaction — autonomous behavior, output quality, and trust. All AX design involves UX thinking, but AX adds a layer of designing for systems that act without direct prompts.

Q: Do I need technical skills to be an AX designer?
A: You need functional literacy with AI systems, not coding expertise. Understanding how LLMs work, what prompts do, and what agents can realistically achieve is essential. But you don't need to build models — you design the experiences around them.

Q: Will AI designers replace AX designers?
A: AI tools will augment AX designers the way digital tools replaced analog ones. The strategic thinking, user empathy, and ethical judgment that AX requires can't be fully automated. AI handles execution; humans handle direction.

Q: What tools do AX designers use?
A: The same core tools as UX designers (Figma, Miro, user testing platforms) plus AI-specific tools: Claude or ChatGPT for research synthesis, agent simulation tools for testing, and prompt design interfaces. The stack is expanding rapidly.

Q: How much do AX designers earn?
A: Roles that embody AX skills command $120,000–$200,000+ in the US. The specific title "AX designer" is rare, but companies are hiring for it under different labels — AI Product Designer, Conversational Designer, Agentic UX Designer.
```

**Brief placeholder:**
```
FAQ [Number]:
Q: [Natural-language question — matches real search queries]
A: [40-80 word answer. Direct. No filler. Citable.]
```
